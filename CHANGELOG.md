### Upgrading from 0.1.0

**0.1.0** was scaffold-only and did not ship a stable exporter API (**[docs/RELEASE_ENGINEERING_SPEC.md](docs/RELEASE_ENGINEERING_SPEC.md) §9.1**). Moving to **0.2.0** is **adoption**, not a rename migration: install `replayt-opentelemetry-exporter`, wire tracer and meter providers per README and **§3** of the public API spec, and wrap each logical run with `workflow_run_span`. There are no canonical metric or lifecycle renames from **0.1.0** because that release did not publish the current instruments or span events.
